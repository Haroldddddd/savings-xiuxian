# 测灵根 · 入门问卷

> 灵根**不是用户挑的，是测出来的**，且**天定不可改**（只有 `init --force` 推倒重来才会重测）。Claude 按本问卷向用户提问、用下面的确定性计分得出灵根，再 `init --root X` 铭刻入道籍。
>
> **核心立意**：测出的亲和品类 = 用户**最容易花超、最该攻克的消费战场**。天道让你在最难的地方修炼，省下来加成最大。这既励志又自洽——不是奖励你已经擅长的，而是点化你直面软肋。

## 灵根 ↔ 品类对照

| 灵根 | 命定战场（亲和品类） |
|------|----------------------|
| 土灵根 | 饮食 🍚 |
| 水灵根 | 出行 🚇 |
| 金灵根 | 购物 🛍️ |
| 火灵根 | 玩乐 🎮 |
| 木灵根 | 居家 🏠 |

## 问卷（3 题，用 AskUserQuestion 一次性问完）

每题 4 个选项，对应 **饮食 / 出行 / 购物 / 玩乐** 四大可压缩战场。第 5 类**居家**(木灵根)走每题自带的"其他"——若用户在"其他"里填了房租/水电/话费/日用/固定账单等，计为**居家**票。

**Q1. 月底回看账单，最让你"肉疼/后悔"的往往是哪类开销？**（最肉疼，权重最高）
- 又在吃上花太多（外卖/下馆子/奶茶咖啡）→ 饮食
- 打车通勤差旅不知不觉花了一堆 → 出行
- 一时冲动剁手买了用不上的东西 → 购物
- 聚会娱乐旅游一玩就刹不住 → 玩乐
- （其他→若填房租水电话费日用等固定开销 → 居家）

**Q2. 哪种场景下你最难管住钱包？**
- 饿的时候 / 嘴馋的时候 → 饮食
- 赶时间 / 懒得挤公共交通时 → 出行
- 看到打折、新品、种草内容时 → 购物
- 朋友一约 / 节假日想犒劳自己时 → 玩乐
- （其他→固定账单类 → 居家）

**Q3. 你最希望这个游戏帮你在哪方面变强？**
- 管住嘴、吃得省 → 饮食
- 优化出行、省下路费 → 出行
- 戒掉冲动购物 → 购物
- 节制娱乐社交开销 → 玩乐
- （其他→把家用账单压下来 → 居家）

## 计分规则（确定性，不靠感觉）

1. 统计 3 题里每个品类被选中的次数（"其他→居家"也计票）。
2. **得票最多的品类 → 对应灵根**。
3. **平票**时，按 **Q1（最肉疼的开销）** 的选择优先——它最能代表真实痛点。若 Q1 不在并列项里，则按并列项中 Q 编号最小的那一题的选择决定。
4. 得出品类后查上表 → `init --root X`（X 为对应灵根的 金/木/水/火/土）。

> 例：Q1饮食、Q2饮食、Q3玩乐 → 饮食 2 票 → 土灵根。
> 例（平票）：Q1出行、Q2购物、Q3饮食 各 1 票 → 按 Q1 取出行 → 水灵根。

## 仪式感（叙事）

- 测前营造"测灵根"的庄重感："道籍未定，需先测你的命定灵根——这将决定你此生在哪条路上修为最快。"
- 测后用脚本输出的灵根+命定战场宣告，强调"天定不可改"。
- 点出励志内核：天道偏要你在最易破财的地方修炼，把软肋炼成战场——在这省下的每一分，修为加成最大。
- **绝不让用户直接点名要哪个灵根**。即便用户说"我想要金灵根"，也婉拒："灵根天定，需如实作答由测试定夺，强求无益。"
